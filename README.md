# Ecommerce Analytics Engineering Project (dbt + PostgreSQL)

This project demonstrates how modern analytics engineering teams build structured data warehouses using **dbt** and **PostgreSQL**.

Raw ecommerce data is transformed into reliable analytics tables using a layered modeling approach.  
The project simulates how companies design transformation pipelines for **revenue analytics, user analytics, and product analytics**.

# Data Warehouse Architecture

The warehouse follows a modern analytics engineering architecture:

RAW DATA

# Transformation Layers

## Staging Layer

Purpose:

- Clean raw data
- Remove duplicates
- Standardize formats
- Prepare data for modeling

Example transformations:

- Deduplication using ROW_NUMBER()
- Null handling using COALESCE
- Currency normalization
- Timestamp standardization

---

## Intermediate Layer

Purpose:

- Join datasets
- Enrich business logic before fact tables

Examples:

int_orders_enriched  
Combines orders with user attributes and acquisition channel.

int_order_refunds  
Aggregates refunds at order level.

---

## Fact Table

### fct_revenue

This is the core revenue model.

It calculates **net revenue per order**.

Net Revenue = Order Amount - Refunded Amount

The model is implemented as an **incremental model** to simulate scalable production pipelines.

# Analytics Data Marts

These models provide **business-ready datasets**.

---

## Monthly Revenue

Model:

mart_monthly_revenue

Tracks total revenue generated per month.

Used for:

- revenue reporting
- growth analysis

---

## User Retention (Cohort Analysis)

Model:

mart_user_retention

Structure:

cohort_month  
order_month  
active_users  

Shows how many users return after signup.

---

## Customer Lifetime Value

Model:

mart_user_ltv

Calculates total revenue generated by each user.

Useful for:

- marketing ROI
- user segmentation

---

## Funnel Conversion

Model:

mart_funnel_conversion

Tracks the user journey across events.

Example funnel:

view → add_to_cart → purchase

Used for:

- product analytics
- conversion optimization
